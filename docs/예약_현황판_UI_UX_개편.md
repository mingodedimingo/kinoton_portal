# 예약 현황판 UI/UX 개편 설계

## 결정 요약

예약 현황판은 외부 캘린더 라이브러리를 새로 추가하지 않고, 현재 프로젝트의 React·Tailwind·date-fns 구성 위에 **경량 커스텀 뷰**로 구현한다. 이 방식은 라이선스 비용이 전혀 없고, 기존 `Monochrome Precision` 디자인 토큰을 그대로 재사용하며, 일간 자원 타임라인처럼 상용 플러그인에 묶이는 기능을 피할 수 있다.

FullCalendar Standard는 MIT 라이선스지만 자원별 타임라인은 Premium 플러그인에 포함된다.[1] [2] FullCalendar Premium의 상업적 사용은 상용 라이선스가 필요하므로 이번 요구사항에는 사용하지 않는다.[1] TOAST UI Calendar는 MIT이며 일간·주간·월간 뷰를 제공하지만, 요청한 ‘세로축 자원·가로축 시간’ 형태를 기존 포털 UX에 맞게 구성하려면 별도 커스터마이징이 많이 필요하다.[3] `react-calendar-timeline`도 MIT이지만 월간·주간 달력까지 함께 해결하지 못하고 추가 의존성과 스타일 통합 비용이 발생한다.[4]

| 대안                    | 라이선스                                | 장점                                         | 제약                                                     | 결정 |
| ----------------------- | --------------------------------------- | -------------------------------------------- | -------------------------------------------------------- | ---- |
| FullCalendar Standard   | MIT                                     | 월간·일반 시간표 구현이 빠름                 | 자원 타임라인은 Premium                                  | 제외 |
| FullCalendar Premium    | 상업 라이선스 또는 제한적 오픈소스 조건 | 완성도 높은 자원 타임라인                    | 사내 상업 환경에서 유료 비용 발생                        | 제외 |
| TOAST UI Calendar       | MIT                                     | 일간·주간·월간 기본 제공                     | 자원 행 기반 일간 뷰와 포털 디자인 통합에 추가 작업 필요 | 제외 |
| react-calendar-timeline | MIT                                     | 자원 타임라인에 적합                         | 주간·월간 달력을 별도로 구현해야 함                      | 제외 |
| React + date-fns 커스텀 | 프로젝트 기존 의존성                    | 비용 없음, 디자인·반응형·예약 모달 연동 최적 | 캘린더 렌더링을 직접 관리                                | 채택 |

## 화면 구조

상단에는 `일간 / 주간 / 월간` 세그먼트 탭, 이전·오늘·다음 날짜 탐색, 현재 범위 제목, 자원 유형 필터, 예약 신청 버튼을 배치한다. 예약 현황은 한 개의 주 카드 안에서 뷰만 전환되며, 예약 데이터는 기존 `reservations.byDateRange`와 `reservationResources.list`를 재사용한다.

| 뷰   | 데스크톱                                                    | 모바일                                                   |
| ---- | ----------------------------------------------------------- | -------------------------------------------------------- |
| 일간 | 왼쪽 고정 자원 열 + 08:00~18:00 시간축, 예약 블록 절대 배치 | 동일 구조를 가로 스크롤하고 자원 열은 고정하여 문맥 유지 |
| 주간 | 자원 행 × 월~일 열, 각 셀에 시간·예약자·목적 요약           | 선택한 요일 한 개를 중심으로 자원별 일정 카드 표시       |
| 월간 | 7열 달력, 날짜 셀에 최대 3건 뱃지와 초과 건수               | 날짜별 예약 건수 중심의 압축 셀 + 선택일 상세 목록       |

## 핵심 상호작용

일간 뷰의 빈 시간대를 클릭하면 해당 자원·날짜·시작 시간이 예약 모달에 자동 입력된다. 시간은 30분 단위로 맞추고 기본 종료 시간은 시작 후 1시간으로 설정한다. 주간 뷰의 빈 날짜 셀과 월간 날짜 셀을 클릭해도 선택한 날짜가 예약 모달에 자동 반영된다.

`대기`와 `승인` 예약은 현황판에 표시하되 색과 라벨을 구분한다. `반려`와 `취소`는 가용 시간을 막지 않으므로 현황판에서 제외하고, 사용자의 ‘내 예약 현황’에서는 기존처럼 상태를 확인할 수 있게 유지한다.

## 접근성과 성능

모든 클릭 가능한 시간 셀과 예약 블록은 키보드 포커스를 받을 수 있게 하고 명확한 `aria-label`을 제공한다. 데이터는 선택된 뷰의 날짜 범위만 조회하며, 월간에서도 최대 약 6주 범위만 요청한다. 외부 캘린더 CSS나 런타임을 추가하지 않아 초기 번들 증가를 최소화한다.
